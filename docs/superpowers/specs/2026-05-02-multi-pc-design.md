## 1. PC 角色定义

| PC | 全名 / 昵称 | 性格关键词 | 独特能力 hook |
|---|---|---|---|
| **昕**（默认） | 罗昕 / 昕 | 微胖吃货 + 社交核心 + 话最多 | 主厨刀伏笔 + 猎奇片+鲁迅 → 对血腥心理抗性高 |
| **砚波** | 砚波 / 砚波 | 最能打 + 危机解析者 + 语文极好 | 高 STR/INT，最先察觉危险 |
| **志勇** | 志勇 / 志勇 | 1.9 米巨汉 + 内心怯 + 二次元宅 + 警察世家 | 高 SIZ/STR + DB +1D6 + act3 警察世家身世 |
| **张怡** | 张怡 / 张怡 | 数学天才 + 魔兽宅 + 满电充电宝 + 神秘学加成 | 高 INT，能"看出"菌丝几何节奏 + 物资 buff |

**全名沿用现状原则：** 砚波/志勇/张怡 没有"姓氏"，PC 时全名=昵称（不编姓，避免设定不一致）。`当前PC名` var 仅在 PC=昕 时为"罗昕"，其他 PC 时等于其昵称。


## 4. 4 PC × 1 Signature Scene（每 PC 不可错过的高光）

### 4.1 昕 — act0 翻衣柜 + 主厨刀 + 厨师梦内心戏

**位置：** act0 寝室傍晚 → 调查菜单 → 翻衣柜（`调查_衣柜` stitch）
**现状：** 主厨刀解锁逻辑已存在
**增量：** PC=昕 + 主厨刀解锁后追加 ~40 行内心戏（厨师梦回响 + 偷藏菜谱本 + 妈妈哭过的回忆）
**实现：** 在 `调查_衣柜` 的"ok"分支末尾加 `{当前PC == 昕}` gate 块
**工作量：** ~80 行（含 condition gate + 主厨刀 ok branch 内的 PC=昕 子分支）

### 4.2 砚波 — act0 夜半叩门 → 早 1 拍触发觉知

**位置：** act0 夜半叩门 第一拍敲门
**现状：** 玩家须 〔聆听 65〕check 通过才推进觉知（门外有东西）
**增量：** PC=砚波 时，**不需要 check**——任何反应选项（大声问/屏息听/让志勇过去看）都触发觉知（门外有东西）+ 一句砚波 PC 内心独白"它不是人。我看了一眼就知道。"
**实现：** 在 3 个反应选项内分别加 `{当前PC == 砚波}` 块覆盖原 check 逻辑
**工作量：** ~60 行

### 4.3 志勇 — act3_boss 阶段 3 警察世家闪回

**位置：** act3_boss 阶段 3（boss 临死前阶段）
**现状：** 阶段 3 narrative 含志勇 NPC 已在战斗中
**增量：** PC=志勇 时触发警察世家闪回 stitch（爸是警察、从小被打到练球、"这种时候你应该顶得住"的回忆）
**实现：** 阶段 3 入口 `{当前PC == 志勇}` 触发 `=== 志勇_警察世家闪回 ===` knot，~80 行
**工作量：** ~80 行

### 4.4 张怡 — act2/act3 走廊几何节奏 → 提前解锁母核位置

**位置：** act2 走廊2F / act3 走廊（菌丝观察场景）
**现状：** act2 已有"菌丝网络贯穿楼栋下水管道"线索
**增量：** PC=张怡 时，走廊调查菜单多一个 `+ 〔神秘学+数学〕看出节奏` 选项 → 计算菌丝几何分布 → 提前解锁母核位置坐标（act3 boss 战可优先攻击该位置 → 减少 boss 战 1 回合）
**实现：** act2 走廊2F 观察菜单加 PC=张怡 条件项 + `=== 张怡_算节奏 ===` 新 knot
**工作量：** ~80 行

**Signature 合计：~300 行**

## 5. 4 PC × 2 Minor 高光

### 5.1 设计原则

- 每 PC 2 段 minor，~30-50 行/段
- 类型：PC-条件解锁的小动作 / 内心独白 / 对其他 NPC 反应
- 不像 signature 那样改变剧情结果，但提供"我玩这个 PC 看到的"独特片段

### 5.2 草案（待 plan 阶段细化）

| PC | Minor 1 | Minor 2 |
|---|---|---|
| **昕** | act0 安抚他人时社交核心特长（〔说服〕成功率 +5）+ 一句吃货段子 | act2 锚点 B 黄兴树（吃货 vs 富二代价值观对话） |
| **砚波** | act1 走廊 危机解析（最先发现菌丝异常）+ 〔聆听〕buff | act2 储藏室 体能优势（撞门/扛物） |
| **志勇** | act0 战斗第一拍（1.9 米巨汉先扑上去 + DB +1D6 加成） | act2 锚点 内心独白：其实我超怕（外强内怯反差） |

**Minor 合计：~12 段 × 40 行 = ~480 行**

## 14. 风险 / Out-of-scope

### 14.1 工作量风险

- 喘息夜 2 工作量比 brainstorm 估算偏多 ~160 行（PC-relative 头尾原计 60，实计 120）—— 已修正记入 §13 总盘子
- 总盘子 ~2670 行，比 brainstorm 估的 ~2280 多 17%
- 主要 buffer 来自 4 PC × 2 minor 高光（§5）和喘息夜 2 PC-relative 头尾（§9.2）

### 14.2 测试 Gate（硬约束）

- ink-only 阶段（PR 1-7）必须**全部 inklecate CLI 编译 clean + 烟测通过**
- 5 endings × 4 PC = 20 路径必须验证（允许妥协方案但不允许跳过测试 gate）
- 测试通过后才进 PR 8（HTML）

### 14.3 itch.io 部署 OUT-OF-SCOPE

- itch.io 部署不在本 spec 范围
- 部署是所有 PR + 全套测试通过后的**独立用户决策**，由用户单独触发
- 部署前置条件：8 PRs 全部 merged + 浏览器手动 playthrough 4 PC × 5 endings 通过

### 14.4 多 PC 中途切换 / 多周目继承（future work）

- 当前 demo 只玩 1 PC × 1 周目；如果未来要支持中途换 PC / 多周目继承，需迁移到前置引擎方案（X 方案，brainstorm Q3 副议题）
- 迁移成本中等：B+E 已经把通用 var 架构做了，X 是在它之上加 JS 引擎层（~150 行 JS 增量）

### 14.5 砚波/志勇/张怡 全名沿用现状

- 砚波/志勇/张怡 没编"姓"，PC 时全名=昵称
- 影响 §12.1 NPC 喊"罗昕" 替换：PC=非昕 时显示昵称（"砚波。" 而不是"罗砚波。"）
- 创作决策保留余地：未来如需正式称呼，可统一编姓（"高砚波" / "马志勇" / "张怡"）
